# ケモホモ作品ファインダー 方針メモ

## 位置づけ

このサイトは「作品を並べるまとめサイト」ではなく、ケモホモ作品をニッチな条件で探せる探索基盤として設計する。

- 入口の言葉は `ケモホモまとめ` を使ってよい
- 商品の本体は `条件検索できるケモホモ作品ファインダー`
- 差別化の核は、DMM 側の粗い分類をそのままなぞることではなく、ユーザー語彙を探索条件へ翻訳すること

## MVP で優先する価値

- 含める条件と除外条件を同じ画面で扱える
- 作品ごとに「なぜこの条件に合うのか」を短く示せる
- 手作り特集を固定導線として持てる
- 管理画面から作品、タグ、特集、公開状態、ログを回せる
- 別テーマへ横展開できるよう、サイトプロファイルを分離しておく

## 入口設計

トップでは次の 3 つをすぐ選べる状態にする。

1. 条件から探す
2. 特集から入る
3. 初見向けの入口条件を見る

公開直後は網羅性より、最初の探索導線の分かりやすさを優先する。

## タグ設計の基本

初期タグは、作品の事実そのものだけでなく「探したい時の言葉」で構成する。

- 入口語: ケモホモ、オスケモ寄り、入口向け
- 作風: やさしめ、軽め、ややシリアス
- 変化要素: TFあり、変化は軽め、精神変化なし
- 関係性: 相棒感、距離が近い、初見でも入りやすい
- 形式: 漫画、CG・イラスト、ノベル
- 運営判断: 手動選抜、導線用
- 除外条件: NTRなし、流血ひかえめ、同意が明確

タグは「作品の属性」と「ユーザーが探す時の言い方」を混ぜて持つ。

## 特集の役割

特集は検索結果の代替ではなく、固定の価値ページとして使う。

- 初見向けの入口
- よく使われる除外条件の受け皿
- TF など深い枝への導線
- SNS から流しやすい着地点

最低限、初見向け、TF入口、除外条件入口、媒体別、雰囲気別の特集を持つ。

## デザイン方針

- 派手な成人向けビジュアルではなく、静かな編集アーカイブ寄り
- 暗緑、琥珀、生成りを基調にした落ち着いたトーン
- 迷わず使える検索 UI を最優先
- 結果カードでは `タイトル / 一言 / 主要タグ / 合う理由` を即読できるようにする
- 露骨な画像に依存せず、文脈で価値を出す

## 管理画面で重視すること

- 作品 CRUD
- 公開 / 下書き / 保留 / 除外
- タグの追加編集と同義語
- 特集の追加編集
- 一括状態変更と一括タグ操作
- 0件検索やよく使われる条件の把握

## 公開前に差し替える項目

- サイト名
- ドメイン
- 問い合わせ窓口
- DMM/FANZA の実リンク
- 免責事項と広告表記
- 実データへの置換

## 実装優先順

1. データモデルをケモホモ向けに寄せる
2. トップ / 検索 / 特集 / 詳細の導線を作る
3. 管理画面に特集運用を追加する
4. ログ可視化を強める
5. 実データ投入前提の差し替え点を README に残す
